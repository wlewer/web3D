## 🎯 测试步骤

### 第1步：调整镜头

1. 页面加载后，你会看到蓝色大闪蝶模型
2. **用鼠标操作**：
   - 左键拖拽：旋转视角
   - 滚轮：缩放
   - 右键拖拽：平移

3. 调整到你认为最佳的观察角度

---

### 第2步：保存镜头

**方式A：点击按钮**
- 点击右下角的 **"💾 保存镜头"** 按钮

**方式B：键盘快捷键**
- 按 `Ctrl + S` (Windows) 或 `Cmd + S` (Mac)

**预期结果**：
- 右上角显示绿色提示："✅ 镜头已保存！"
- 右侧面板显示："✅ 已保存配置"

---

### 第3步：切换模型

1. 在右侧面板的"选择模型"区域
2. 点击 **"🐱 可爱猫咪"** 或按数字键 `2`
3. 观察模型切换

---

### 第4步：验证自动恢复

1. 再次点击 **"🦋 蓝色大闪蝶"** 或按数字键 `1`
2. **观察**：相机会自动恢复到刚才保存的视角！
3. 右侧面板显示："✅ 已保存配置"

---

### 第5步：重置镜头

**方式A：点击按钮**
- 点击 **"🔄 重置镜头"** 按钮

**方式B：键盘快捷键**
- 按 `Ctrl + R` (Windows) 或 `Cmd + R` (Mac)

**预期结果**：
- 相机回到智能居中的默认视角
- 右侧面板显示："⚪ 使用默认视角"

## 🔍 调试技巧

### 1. 查看控制台日志

打开浏览器开发者工具（F12），切换到Console标签，你会看到：

```javascript
// 保存时
📸 保存相机配置: { position: [...], target: [...], zoom: 1 }
💾 保存的配置: { position: [...], target: [...], zoom: 1 }

// 加载时
📥 检测到自定义相机配置，自动应用...
✅ 自定义相机配置应用完成

// 重置时
🔄 重置相机到默认位置
✅ 相机已重置
```

---

### 2. 查看localStorage数据

在Console中执行：
```javascript
// 查看所有保存的配置
console.log(JSON.parse(localStorage.getItem('test-camera-configs')));

// 查看单个模型的配置
const configs = JSON.parse(localStorage.getItem('test-camera-configs'));
console.log(configs['butterfly']);
```

---

### 3. 手动修改配置

```javascript
// 获取当前配置
const configs = JSON.parse(localStorage.getItem('test-camera-configs'));

// 修改蝴蝶模型的相机位置
configs['butterfly'].position = [5, 3, 5];

// 保存回去
localStorage.setItem('test-camera-configs', JSON.stringify(configs));

// 刷新页面，观察变化
location.reload();
```

## ⚠️ 常见问题

### Q1: 保存后刷新页面，配置丢失了？

**检查**：
1. 打开Console，查看是否有错误
2. 检查localStorage是否有数据：
   ```javascript
   localStorage.getItem('test-camera-configs')
   ```

**可能原因**：
- 浏览器禁用了localStorage
- 使用了隐私模式/无痕模式
- localStorage容量已满

---

### Q2: 切换模型后，没有自动恢复视角？

**检查**：
1. 确认该模型之前确实保存过配置
2. 查看右侧面板是否显示"✅ 已保存配置"
3. 查看Console日志是否有错误

**可能原因**：
- 模型还在加载中，配置应用时机不对
- 配置数据格式不正确

---

### Q3: 保存按钮点击没反应？

**检查**：
1. 确认模型已经完全加载
2. 查看Console是否有错误信息
3. 确认cardRef.current不为null

**调试**：
```javascript
// 在Console中执行
console.log(cardRef.current); // 应该不是null
```

---

### Q4: 重置后视角不对？

**说明**：
- 重置会调用智能居中算法
- 每次重置的结果可能略有不同（取决于模型包围盒计算）
- 这是正常现象
