## WebGL Context 耗尽修复

**问题**：控制台出现大量 `THREE.WebGLRenderer: Context Lost` + `Model3DCard: WebGL Context Lost`，根因是 `Model3DCard` 组件在列表渲染时每个卡片都无条件创建独立 WebGL Context，超出浏览器上限（8~16）。

**修复内容**：
1. `Model3DCard.tsx`：将 WebGL 初始化条件改为 `if (!isActive) return`，默认显示静态缩略图占位（`model-thumbnail-placeholder`），点击激活后才创建 3D renderer；cleanup 时完整销毁资源并置 null
2. `Model3DCard.css`：补充 `.model-thumbnail-placeholder` 和 `.thumbnail-hover-hint` 样式
3. `SparkViewer.tsx`：完善 cleanup 逻辑，sparkRef/rendererRef/controlsRef/sceneRef 均置 null，确保切换模型时 WebGL Context 被正确释放
