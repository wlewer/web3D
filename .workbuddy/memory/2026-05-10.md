## ThreepipeEditorPage 翻译问题修复（第三次）

**文件：** `src/web-frontend/src/admin/modules/experimental/pages/ThreepipeEditorPage.tsx`

### 问题现象
切换中文后，Tab 标签（Viewer/Interaction/Buffers...）和面板内容仍为英文。

### 根本原因（经代码分析）
1. **Tab 标签名来自插件类名**（如 `ViewerUiConfigPlugin` → `"Viewer"`），渲染为 DOM `.mode-button` 元素
2. **`translateUiConfig` 只处理 `uiConfig.label`**（JS 对象树），无法触及已渲染的 DOM 标签
3. **原 `loadPlugins` 用法错误** — 传了 `{ "查看器": [plugins] }` 对象，但正确 API 是插件类数组 `[ViewerUiConfigPlugin, ...]`

### 修复内容
1. **修正 `loadPlugins` 调用** — 改为直接传插件类数组（正确 API）
2. **`translateUiConfig` 加入调试日志** — 控制台打印 `✅ 已翻译` 和 `❌ 未找到翻译` 的 label
3. **新增 DOM 翻译逻辑** — `setTimeout 400ms` 后遍历 `.mode-buttons-container .mode-button`，用 `CATEGORY_LABELS` 映射替换 Tab 文字
4. **语言切换按钮改为始终显示** — 从 `{!initializing && ...}` 条件渲染改为直接渲染，避免初始化慢时按钮永不出现
