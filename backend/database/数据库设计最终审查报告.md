#### 2.1 索引类型分布

| 索引类型 | 数量 | 用途 | 示例 |
|---------|------|------|------|
| **B-tree索引** | 40+ | 等值查询、范围查询 | idx_users_email |
| **GIN索引** | 8 | 数组搜索、JSONB查询 | idx_models_tags |

| **复合索引** | 5 | 多字段联合查询 | idx_versions_number (model_id, version_number) |
| **唯一索引** | 10+ | 防止重复数据 | UNIQUE(user_id, model_id) |
| **降序索引** | 4 | 加速排序 | idx_models_created_at DESC |

### 10. 潜在风险与缓解措施 ✅

#### 10.1 已识别风险

| 风险 | 严重程度 | 缓解措施 | 状态 |
|------|---------|---------|------|
| JSONB字段过大影响性能 | 中 | 限制JSONB大小<1MB，使用压缩 | ✅ 已考虑 |
| 分区表需要定期维护 | 低 | 自动化脚本创建新分区 | ✅ 已规划 |
| GIN索引占用空间大 | 低 | 监控磁盘使用，定期VACUUM | ✅ 已考虑 |
| 外键级联删除误操作 | 中 | 应用层二次确认，软删除优先 | ✅ 已实施 |
| 并发写入冲突 | 低 | PostgreSQL MVCC机制，乐观锁 | ✅ 已保障 |

#### 10.2 监控建议

```sql
-- 慢查询监控
SELECT * FROM pg_stat_statements 
ORDER BY total_exec_time DESC 
LIMIT 10;

-- 索引使用情况
SELECT schemaname, tablename, indexname, idx_scan 
FROM pg_stat_user_indexes 
ORDER BY idx_scan ASC;

-- 表大小监控
SELECT 
    schemaname,
    tablename,
    pg_size_pretty(pg_total_relation_size(schemaname || '.' || tablename)) AS size
FROM pg_tables
WHERE schemaname = 'public'
ORDER BY pg_total_relation_size(schemaname || '.' || tablename) DESC;

-- 连接数监控
SELECT count(*) FROM pg_stat_activity;
```
