### 7. 审计日志系统（Audit Logs）✅ 分区表优化

#### 7.1 audit_logs（操作审计日志）

**分区表设计**：

```sql
CREATE TABLE audit_logs (
    id UUID PRIMARY KEY,
    user_id UUID,
    action VARCHAR(100),              -- login/create_model/update_page等
    resource_type VARCHAR(50),
    resource_id UUID,
    
    ip_address INET,
    user_agent TEXT,
    request_method VARCHAR(10),
    request_path TEXT,
    
    request_data JSONB,
    response_data JSONB,
    
    status VARCHAR(20) CHECK (status IN ('success', 'failed')),
    status_code INTEGER,
    error_message TEXT,
    
    created_at TIMESTAMP WITH TIME ZONE
) PARTITION BY RANGE (created_at);

-- 按月分区
CREATE TABLE audit_logs_2025_04 PARTITION OF audit_logs
    FOR VALUES FROM ('2025-04-01') TO ('2025-05-01');
```

**优势**：
- ✅ 高性能查询（只扫描相关月份分区）
- ✅ 易于归档和清理旧数据
- ✅ 满足合规要求（GDPR等）

**使用场景**：
- 安全审计（谁在什么时候做了什么）
- 问题排查（API调用失败原因）
- 用户行为分析

#### 7.2 api_call_logs（API调用日志）

```sql
CREATE TABLE api_call_logs (
    endpoint VARCHAR(255),
    method VARCHAR(10),
    status_code INTEGER,
    response_time_ms INTEGER,         -- 响应时间（毫秒）
    ip_address INET,
    created_at TIMESTAMP
) PARTITION BY RANGE (created_at);
```

**用途**：
- API性能监控

-- GIN索引（数组/JSONB搜索）
CREATE INDEX idx_models_tags ON models_3d USING GIN(tags);
CREATE INDEX idx_models_name_i18n ON models_3d USING GIN(name_i18n);

-- 复合索引
CREATE INDEX idx_versions_number ON model_versions(model_id, version_number DESC);
CREATE INDEX idx_sections_position ON page_sections(page_id, position);

-- 降序索引
CREATE INDEX idx_models_created_at ON models_3d(created_at DESC);
