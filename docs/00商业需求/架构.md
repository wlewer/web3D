核心设计原则：

多租户隔离：每个官网独立数据库 schema 或使用 tenant_id 行级隔离。

Headless 优先：前后端彻底分离，所有页面/组件数据通过 API 获取，便于未来扩展小程序/APP。

插件化架构：官网引擎本身只是一个“壳”，所有功能（3D 查看器、轮播、表单、AI 生成入口）都是可插拔的组件。

2.2 插件化实现机制（基于现有 TemplateRenderer 升级）
typescript
// 组件注册表核心接口
interface IComponentPlugin {
  id: string;                     // 全局唯一
  name: string;
  category: ComponentCategory;
  version: string;
  
  // 渲染相关
  renderer: React.ComponentType<ComponentProps>;
  editor?: React.ComponentType<EditorProps>;  // 可视化编辑器
  defaultConfig: Record<string, any>;
  
  // 生命周期钩子
  onMount?: (element: HTMLElement, config: any) => void;
  onUpdate?: (config: any) => void;
  onUnmount?: () => void;
  
  // 依赖声明
  dependencies?: {
    scripts?: string[];   // 动态加载 external lib
    styles?: string[];
    components?: string[]; // 依赖其他组件
  };
}

// 插件管理器实现
class ComponentRegistry {
  private components = new Map<string, IComponentPlugin>();
  
  register(plugin: IComponentPlugin) {
    // 检查版本冲突、循环依赖等
    this.components.set(plugin.id, plugin);
    // 动态加载依赖脚本
    if (plugin.dependencies?.scripts) {
      plugin.dependencies.scripts.forEach(src => this.loadScript(src));
    }
  }
  
  getRenderer(id: string): React.ComponentType {
    return this.components.get(id)?.renderer || FallbackComponent;
  }
}

⚡ 三、性能优化体系：让 3D 官网秒开、流畅
3.1 加载策略（分级加载）
级别	触发条件	加载内容
L0 (骨架屏)	页面请求	HTML 结构 + CSS 占位
L1 (首屏必需)	进入视口	Hero 区域的 3D 模型（低精度版本）、UI 文案
L2 (次屏)	滚动到附近	其他 3D 模型、高清纹理
L3 (交互时)	用户点击/悬停	完整配置器、高精度 LOD、动画
实现技术：

IntersectionObserver 监听组件可见性

模型预加载队列：new THREE.LoadingManager() + 优先级调度

SPZ 渐进式渲染：已支持，继续优化首帧时间

3.2 3D 模型交付管线
text
用户上传/生成 → 后端自动优化 → 多版本输出 → CDN 分发

优化步骤：
1. 减面（Blender 或 trimesh）：保持视觉基本不变，面数减少 50%~80%
2. 纹理压缩（basisu / KTX2）：体积减少 70%
3. 生成 LOD 链：model_high.glb, model_medium.glb, model_low.glb
4. 转换为 SPZ（如果适用）
5. 上传到 MinIO + CDN（如 CloudFront 或 又拍云）
3.3 运行时性能监控
帧率仪表盘：当掉帧超过阈值（如 <30fps），自动调低 LOD 级别或降低阴影质量。

内存监控：Three.js 的 renderer.info.memory，防止大模型导致 OOM。

网络监控：慢模型请求自动降级为 fallback 模型或提示。

4.1 动态页面渲染器（TemplateRenderer 升级）
tsx
// 核心渲染组件
const DynamicPage: React.FC<{ pageId: string }> = ({ pageId }) => {
  const { data: pageSchema } = useQuery(['page', pageId], fetchPageSchema);
  const components = useComponentRegistry();
  
  // 递归渲染嵌套组件
  const renderComponent = (comp: ComponentSchema) => {
    const Comp = components.getRenderer(comp.type);
    return (
      <Comp key={comp.id} {...comp.props}>
        {comp.children?.map(child => renderComponent(child))}
      </Comp>
    );
  };
  
  return <div className="dynamic-page">{renderComponent(pageSchema.root)}</div>;
};
4.2 3D 模型数据绑定
组件可以从多种来源获取模型：

直接引用：从后台模型库选择一个已上传的模型 ID。

动态 URL：通过 API 返回的模型列表渲染轮播。

6.2 自动化流水线
GitHub Actions / GitLab CI：提交代码 → 单元测试 → 构建镜像 → 推送镜像仓库 → 自动部署到测试环境 → 集成测试 → 生产环境（需审批）。

蓝绿发布：降低升级风险。

6.3 成本优化建议
模型存储冷热分离：高频访问模型放 CDN，冷模型仅存 MinIO。

GPU 节点使用竞价实例：用于 AI 生成任务，可节省 60% 成本。

无服务化边缘：对于纯展示型页面，考虑使用 CloudFlare Workers + R2 全静态化。
