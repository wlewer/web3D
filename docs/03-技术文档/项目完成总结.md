## 🎯 核心特性

### 1. 三模式架构

| 模式 | 用途 | 依赖 | 启动时间 |
|------|------|------|---------|
| **Mock** | 技术演示 | 无 | 立即 |
| **Local** | 真实生成 | GPU + Hunyuan3D | ~10分钟部署 |
| **Cloud** | 生产环境 | API Key | 立即 |

### 2. 完整的技术流程

```
用户上传图片
    ↓
前端调用API
    ↓
后端接收请求
    ↓
根据模式选择处理方式
    ↓
返回任务ID
    ↓
前端轮询状态
    ↓
显示进度条
    ↓
生成完成后提供下载
```

## 🚀 使用方法

### 方式一：一键启动（推荐）

```bash
# Windows
start_demo.bat

# Mac/Linux
chmod +x start_demo.sh
./start_demo.sh
```

### 方式二：手动启动

```bash
# 终端1：启动后端
cd backend
python -m uvicorn app.main:app --reload --host 0.0.0.0 --port 8000

# 终端2：启动前端
cd src/web-frontend
npm run dev
```

### 访问地址

- 本地：http://localhost:5173
- 局域网：http://[你的IP]:5173
- API文档：http://localhost:8000/docs

## 💡 关键决策说明

### 为什么选择三模式架构？

1. **降低入门门槛**：Mock模式无需任何依赖
2. **灵活性强**：可根据需求选择模式
3. **易于演示**：快速展示技术可行性
4. **平滑过渡**：从演示到生产的自然演进

### 为什么Mock模式默认5秒？

1. **足够展示流程**：让用户看到进度变化
2. **不会太长**：避免等待焦虑
3. **可配置**：可根据需要调整

### 为什么前端不关心模式？

1. **关注点分离**：前端只负责UI交互
2. **降低耦合**：后端可自由切换模式
3. **易于维护**：前端代码更简洁
