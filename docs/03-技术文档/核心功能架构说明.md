## 📊 核心功能流程

### 3D 生成流程
```
用户上传 2D 图片
    ↓
前端调用 POST /api/v1/generation/upload
    ↓
后端接收图片并编码为 base64
    ↓
调用 Hunyuan3D API (端口 8081)
    ↓
Hunyuan3D 生成 3D 模型
    ↓
返回任务 ID (uid)
    ↓
前端轮询查询状态 (每 3 秒)
    ↓
生成完成，下载 GLB 文件
    ↓
保存到数据库和文件系统
    ↓
前端使用 Spark 2.0 渲染展示
```

### 后台管理流程
```
管理员登录 (/admin/login)
    ↓
JWT 认证
    ↓
访问管理模块
    ↓
通过 Refine Data Provider 调用后端 API
    ↓
显示数据列表/详情
    ↓
执行 CRUD 操作
    ↓
更新数据库
```

## ⚠️ 注意事项

### 1. Hunyuan3D 服务
- 当前使用 **Mock 模式**（因为 Hunyuan3D 服务未启动）
- 真实部署需要 GPU 服务器或云端服务
- Mock 模式下会返回模拟的 3D 模型数据

### 2. 数据库
- 当前使用 **SQLite** (开发环境)
- 生产环境建议切换到 **PostgreSQL**

### 3. 对象存储
- 当前使用 **Mock Storage** (本地文件系统)
- 生产环境建议切换到 **MinIO** 或 **阿里云 OSS**

### 4. 环境变量
- 前端 `.env` 文件配置了 `VITE_API_BASE_URL=http://localhost:8000`
- 确保后端运行在 8000 端口
