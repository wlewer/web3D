# 三模式架构设计说明

## 🎯 设计理念

本项目的核心设计目标是：**从演示到生产的平滑过渡**。

通过实现三种运行模式，我们可以在不同阶段使用不同的技术方案，避免"一步到位"带来的复杂性。

## 📊 三种模式对比

### 1. Mock模式（演示模式）

**特点：**
- ✅ 无需任何外部依赖
- ✅ 立即启动，零配置
- ✅ 模拟完整流程
- ❌ 不生成真实3D模型

**适用场景：**
- 技术演示
- 手机端测试
- 前端UI验证
- API接口联调

**实现方式：**
```python
async def _mock_generate(self, image_path: str) -> dict:
    mock_uid = f"mock_{int(time.time())}"
    return {
        "uid": mock_uid,
        "status": "processing",
        "glb_path": None
    }

async def _mock_check_status(self, uid: str) -> dict:
    # 5秒后自动完成
    if int(time.time()) - start_time >= 5:
        return {"status": "completed"}
    return {"status": "processing"}
```

**配置：**
```env
HUNYUAN3D_MODE=mock
```

---

### 2. Local模式（本地GPU服务）

**特点：**
- ✅ 真实AI生成
- ✅ 数据隐私安全
- ✅ 无网络依赖
- ❌ 需要NVIDIA GPU
- ❌ 部署复杂度高

**适用场景：**
- 本地开发测试
- 私有化部署
- 对数据隐私要求高

**实现方式：**
```python
async def _local_generate(self, image_path: str, ...) -> dict:
    # 1. 读取图片并转为base64
    with open(image_path, 'rb') as f:
        image_base64 = base64.b64encode(f.read()).decode('utf-8')
    
    # 2. 调用本地Hunyuan3D API
    response = await self.client.post(
        f"{self.base_url}/send",
        json={"image": image_base64, ...}
    )
    
    return {"uid": response.json()["uid"], "status": "processing"}
```

**配置：**
```env
HUNYUAN3D_MODE=local
HUNYUAN3D_BASE_URL=http://localhost:8081
```

**部署要求：**
- NVIDIA GPU (RTX 3090+)
- CUDA 11.8+
- Hunyuan3D服务已部署

---

### 3. Cloud模式（腾讯云服务）

**特点：**
- ✅ 真实AI生成
- ✅ 无需GPU硬件
- ✅ 易于扩展
- ❌ 需要API Key
- ❌ 有网络延迟
- ❌ 可能产生费用

**适用场景：**
- 生产环境
- 无GPU服务器
- 需要高可用性

**实现方式：**
```python
async def _cloud_generate(self, image_path: str, ...) -> dict:
    # TODO: 调用腾讯云API
    headers = {"Authorization": f"Bearer {self.cloud_api_key}"}
    response = await self.client.post(
        "https://hy3d.dev/api/generate",
        headers=headers,
        files={"image": open(image_path, 'rb')}
    )
    return response.json()
```

**配置：**
```env
HUNYUAN3D_MODE=cloud
HUNYUAN3D_CLOUD_API_KEY=your_api_key_here
```

## 🔄 模式切换流程

### 从Mock切换到Local

```bash
# 1. 停止当前服务
# Ctrl+C 停止后端

# 2. 编辑配置文件
cd backend
cp .env.example .env
# 修改 HUNYUAN3D_MODE=local

# 3. 部署Hunyuan3D服务
git clone https://github.com/Tencent/Hunyuan3D-1.git
cd Hunyuan3D-1
pip install -r requirements.txt
python api_server.py --host 0.0.0.0 --port 8081

# 4. 重启后端服务
python -m uvicorn app.main:app --reload
```

### 从Local切换到Cloud

```bash
# 1. 编辑配置文件
cd backend
# 修改 .env
HUNYUAN3D_MODE=cloud
HUNYUAN3D_CLOUD_API_KEY=your_api_key

# 2. 重启后端服务
# Ctrl+C 停止，然后重新启动
```

## 🎨 前端适配

前端代码**无需关心**后端使用哪种模式，因为：

1. **统一的API接口**
   - 所有模式使用相同的REST API
   - 返回格式一致

2. **透明的状态管理**
   - 前端只关注任务ID和状态
   - 不关心后端如何实现

3. **友好的用户提示**
   - 页面顶部显示当前模式
   - Mock模式显示警告信息

```tsx
<Alert
  message="当前运行模式"
  description={
    <div>
      <p>• <strong>Mock模式</strong>：用于本地演示和技术验证（当前）</p>
      <p>• <strong>Local模式</strong>：需要部署本地GPU服务</p>
      <p>• <strong>Cloud模式</strong>：调用腾讯官方云服务</p>
    </div>
  }
  type="info"
/>
```

## 🔧 技术细节

### 环境变量优先级

```
函数参数 > 环境变量 > 默认值
```

```python
def get_hunyuan3d_service(mode=None, base_url=None, cloud_api_key=None):
    # 1. 优先使用函数参数
    if mode is None:
        # 2. 其次使用环境变量
        mode = os.getenv("HUNYUAN3D_MODE", "mock")
    
    if base_url is None:
        base_url = os.getenv("HUNYUAN3D_BASE_URL", "http://localhost:8081")
    
    if cloud_api_key is None:
        cloud_api_key = os.getenv("HUNYUAN3D_CLOUD_API_KEY")
    
    # 3. 创建服务实例
    return Hunyuan3DService(mode=mode, base_url=base_url, ...)
```

### 单例模式

```python
_hunyuan_service_instance: Optional[Hunyuan3DService] = None

def get_hunyuan3d_service(...) -> Hunyuan3DService:
    global _hunyuan_service_instance
    
    if _hunyuan_service_instance is None:
        _hunyuan_service_instance = Hunyuan3DService(...)
    
    return _hunyuan_service_instance
```

### 健康检查优化

```python
# Mock模式跳过健康检查
if service.mode != "mock":
    try:
        health_check = await service.client.get(f"{service.base_url}/health")
        if health_check.status_code != 200:
            raise Exception("Service not healthy")
    except Exception:
        # 返回Mock响应，避免阻塞测试
        return mock_response()
```

## 📈 扩展性设计

### 未来可以添加的模式

1. **Hybrid模式**
   - 优先使用Local模式
   - Local失败时自动切换到Cloud
   - 提高可用性

2. **Queue模式**
   - 将任务放入消息队列
   - Celery/RQ异步处理
   - 支持高并发

3. **Cache模式**
   - 缓存相似图片的生成结果
   - 减少重复计算
   - 提高响应速度

### 如何添加新模式

```python
class Hunyuan3DService:
    async def generate_from_image(self, ...):
        if self.mode == "mock":
            return await self._mock_generate(...)
        elif self.mode == "local":
            return await self._local_generate(...)
        elif self.mode == "cloud":
            return await self._cloud_generate(...)
        elif self.mode == "new_mode":  # 新增
            return await self._new_mode_generate(...)  # 实现新方法
        else:
            raise ValueError(f"Unknown mode: {self.mode}")
```

## 🎯 最佳实践

### 1. 开发阶段

- 使用 **Mock模式** 快速验证前端逻辑
- 无需等待AI生成，提高开发效率
- 专注于UI/UX优化

### 2. 测试阶段

- 切换到 **Local模式** 测试真实生成效果
- 验证图片质量要求
- 测试不同物体的生成效果

### 3. 生产环境

- 根据成本选择 **Local** 或 **Cloud** 模式
- 监控生成成功率和耗时
- 收集用户反馈优化

### 4. 演示场景

- 使用 **Mock模式** 确保稳定性
- 提前准备高质量示例图片
- 强调这是"技术演示"而非最终产品
