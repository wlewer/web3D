## ❓ 优化2：是否支持多图片处理？

### **当前实现：仅支持单图** 🔴

#### **后端API定义**

```python
@router.post("/huggingface/upload")
async def upload_huggingface(
    file: UploadFile = File(...),  # ← 单个文件
    version: str = Form(default="hy-3d-3.0"),
    ...
):
```

**关键点**：
- `file: UploadFile = File(...)` - 只接收**一个文件**
- 不支持 `List[UploadFile]` 或 `files: UploadFile = File(...)`

---

#### **前端实现**

```typescript
<Upload
  maxCount={1}  // ← 限制为1个文件
  ...
>
```

**关键点**：
- `maxCount={1}` - 明确限制只能选择1张图片
- 不支持多图选择

---

### **腾讯混元3D API限制**

根据腾讯官方文档：
> **输入要求**：单张RGB图像（PNG/JPG格式）

**不支持**：
- ❌ 多张图片同时输入
- ❌ 多角度图片融合
- ❌ 视频帧序列

**原因**：
- 混元3D是**单图到3D**模型生成
- 每张图独立生成一个3D模型
- 不支持多图联合推理

---

### **如果需要多图处理怎么办？**

#### **方案1：批量提交任务**（推荐）

```typescript
// 用户选择多张图片
const files = [file1, file2, file3];

// 循环提交
for (const file of files) {
  await submitGeneration(file);
  // 每次生成一个模型
}
```

**优势**：
- ✅ 简单直接
- ✅ 每个模型独立生成
- ✅ 失败不影响其他任务

**劣势**：
- ❌ 耗时长（每个30-60秒）
- ❌ 费用高（每个10-20积分）

---

#### **方案2：多图拼接**（不推荐）

```
图片1    图片2    图片3
  ↓        ↓        ↓
拼接成一张大图
  ↓
输入混元3D
  ↓
生成一个3D模型（质量差）
```

**问题**：
- ❌ 模型质量差
- ❌ 不符合API设计
- ❌ 结果不可控

---

#### **方案3：使用其他AI模型**（高级）

有些模型支持多图输入：

| 模型 | 多图支持 | 说明 |
|------|---------|------|
| **混元3D** | ❌ 不支持 | 单图到3D |
| **TripoSR** | ❌ 不支持 | 单图到3D |
| **InstantMesh** | ❌ 不支持 | 单图到3D |
| **Zero123++** | ✅ 支持 | 多角度到3D |
| **LRM** | ✅ 支持 | 多图融合 |

**注意**：这些模型需要本地GPU部署，不适合当前项目。

// 使用队列管理多个任务
class TaskQueue {
  private queue: Task[] = [];
  private running: boolean = false;
  
  async add(task: Task) {
    this.queue.push(task);
    this.process();
  }
  
  async process() {
    if (this.running || this.queue.length === 0) return;
    this.running = true;
    
    const task = this.queue.shift();
    await executeTask(task);
    
    this.running = false;

  }
