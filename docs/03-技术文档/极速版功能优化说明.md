# 极速版功能优化说明

> **优化时间**：2026-04-18  
> **优化原因**：避免误导用户，保持与官方一致性

---

## 📋 问题背景

### 1. 官方版本确认

通过查阅腾讯混元3D官方文档（https://cloud.tencent.com/document/product/1804/120696），确认：

✅ **腾讯官方确实存在"极速版"（HY-3D-Express）**
- 生成时间：10-20秒
- 模型质量：中等（85分）
- 适用场景：快速原型验证、批量生成

❌ **但腾讯云Python SDK尚未支持该版本的API接口**
- 缺少 `SubmitHunyuanTo3DExpressJob` 方法
- 缺少 `QueryHunyuanTo3DExpressJob` 方法
- SDK仅支持标准版（rapid）和专业版（pro）

### 2. 之前的问题

之前的实现方案是**降级使用标准版**，但这会导致：
- ❌ 用户体验不一致：选择极速版但实际使用标准版
- ❌ 生成时间不符：预期10-20秒，实际30-60秒
- ❌ 可能引起误解：用户以为系统有问题

---

## ✅ 优化方案

### 核心原则

**保持与官方一致性，避免误导用户**

既然SDK不支持，就明确告知用户"即将开放"，而不是偷偷降级。

### 具体实现

#### 1. **前端UI优化**

##### (1) 模型卡片顶部栏
- ✅ 添加"即将开放"标签（灰色徽章）
- ✅ 降低透明度至0.5，视觉上表示停用
- ✅ 虚线边框（`border: 1px dashed #d9d9d9`）
- ✅ 鼠标悬停显示"not-allowed"光标
- ✅ 点击时弹出提示："该模型版本即将开放，敬请期待！"

##### (2) 下拉选择器
- ✅ Option组件设置 `disabled={true}`
- ✅ 标签旁显示"即将开放"小标签
- ✅ 用户无法选中该选项

##### (3) 开始生成按钮
- ✅ 检测当前模型是否停用
- ✅ 停用时按钮显示"🔜 即将开放"
- ✅ 按钮背景色为灰色（`#f0f0f0`）
- ✅ 图标改为信息图标（`InfoCircleOutlined`）
- ✅ 按钮完全禁用（`disabled={true}`）

## 🔄 后续启用流程

当腾讯云SDK更新支持极速版后，只需以下3步即可启用：

### Step 1: 后端SDK升级
```bash
pip install --upgrade tencentcloud-sdk-python
```

### Step 2: 修改配置文件
编辑 `ProfessionalGenerationPage.tsx` 第133-168行：
```typescript
{
  id: 'HY-3D-Express',
  icon: '⚡',
  label: 'HY-3D-Express 极速版',
  desc: '最快生成速度，适合快速原型',  // 恢复描述
  version: 'HY-3D-Express',
  badges: [
    { text: '极速', color: 'orange' },  // 恢复橙色标签
    { text: '云端', color: 'blue' },
  ],
  endpoint: '/api/v1/experimental/huggingface/upload',
  costPerUse: 5,
  estimatedTime: '10-20秒',
  tokenCost: '~25,000 Tokens/次',
  disabled: false,  // ← 改为 false
  disableReason: undefined,  // ← 清空原因
  details: {
    title: '⚡ HY-3D-Express 极速版',  // 移除"（即将开放）"
    // ... 其他配置保持不变
    recommended: '🎯 追求速度的首选，适合快速迭代场景',  // 恢复推荐语
  },
}
```

### Step 3: 重启前端服务
```bash
cd src/web-frontend
npm run dev
```

**无需修改后端代码**，因为后端已有降级逻辑保护，SDK支持后会自动生效。

## 💡 设计原则总结

### 1. **诚实透明**
- 不隐瞒技术限制
- 明确告知用户真实情况
- 建立长期信任关系

### 2. **用户体验优先**
- 避免用户产生错误期望
- 提供清晰的视觉反馈
- 友好的提示信息

### 3. **易于维护**
- 简单的配置开关
- 清晰的代码结构
- 便于未来扩展

### 4. **与官方一致**
- 尊重官方发布节奏
- 不自行实现未支持的接口
- 等待官方SDK更新
