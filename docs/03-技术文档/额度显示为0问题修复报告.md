## 🔍 问题根本原因

### **原因1：Settings配置类缺少HUNYUAN3D_MODE字段** 🔴

**错误信息**：
```python
pydantic_core._pydantic_core.ValidationError: 1 validation error for Settings
HUNYUAN3D_MODE
  Extra inputs are not permitted [type=extra_forbidden, input_value='mock', input_type=str]
```

**原因**：
- `.env`文件中定义了`HUNYUAN3D_MODE=mock`
- 但`backend/app/config.py`的`Settings`类中没有这个字段
- Pydantic默认不允许额外字段，导致启动失败
- 后端服务无法正常运行，API返回错误

**影响**：
- ❌ 后端启动失败或运行时错误
- ❌ 额度API无法正常调用
- ❌ 前端显示额度为0

---

### **原因2：testuser用户缺少额度记录** 🟡

**症状**：
- admin用户：有额度记录（200积分）
- testuser用户：无额度记录

**原因**：
- 用户创建时没有自动初始化额度记录
- 需要手动或通过API触发创建

## ❓ 常见问题

### **Q1: 为什么之前额度是正常的，现在变成0了？**

**A**: 可能的原因：
1. 后端重启时加载配置失败（ValidationError）
2. 数据库中的额度记录被意外删除
3. 用户session过期，加载了错误的用户数据

**解决**：运行`check_and_fix_quota.py`脚本检查和修复

---

### **Q2: 如何给特定用户增加额度？**

**A**: 方法1 - 使用API：
```bash
# 调用管理员API
curl -X PUT "http://localhost:8000/api/v1/quota/admin/update-package/{user_id}" \
  -H "Authorization: Bearer {admin_token}" \
  -H "Content-Type: application/json" \
  -d '{"package_type": "pro", "total_quota": 500}'
```

方法2 - 直接修改数据库：
```sql
UPDATE user_quotas 
SET total_quota = 500, remaining_quota = 500 - used_quota 
WHERE user_id = '{user_id}';
```

---

### **Q3: 额度用完了怎么办？**

**A**: 
1. **临时方案**：运行`check_and_fix_quota.py`重置额度
2. **正式方案**：实现充值/购买额度功能
3. **管理方案**：管理员通过API调整用户额度

---

### **Q4: Mock模式和Cloud模式的额度有什么区别？**

**A**: 
| 模式 | 额度扣除 | 额度退还 | 说明 |
|------|---------|---------|------|
| **Mock** | ✅ 扣除 | ✅ 失败时退还 | 用于测试额度管理逻辑 |
| **Cloud** | ✅ 扣除 | ✅ 失败时退还 | 真实调用API，消耗实际费用 |

**注意**：两种模式都会扣除和退还额度，Mock模式只是为了测试流程。

---

### **Q5: 如何查看额度使用历史？**

**A**: 查看数据库表：
```sql
SELECT * FROM quota_usage_logs 
WHERE user_id = '{user_id}' 
ORDER BY created_at DESC;
```

或实现额度使用记录API（待开发）。
