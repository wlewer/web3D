# 专业版3D生成功能 - 快速测试指南

> **测试时间**：5-10分钟  
> **测试目标**：验证模型能够真实生成并显示在右侧面板

---

## 🚀 快速开始（Mock模式）

### Step 1: 确认配置

检查 `.env` 文件：
```bash
HUNYUAN3D_MODE=mock
```

✅ 如果是 `mock`，继续下一步  
❌ 如果是 `cloud`，改为 `mock` 并重启后端

---

### Step 2: 启动服务

**终端1 - 后端**：
```bash
cd backend
python -m uvicorn app.main:app --reload --port 8000
```

等待看到：
```
INFO:     Application startup complete.
INFO:     Uvicorn running on http://127.0.0.1:8000
```

**终端2 - 前端**：
```bash
cd src/web-frontend
npm run dev
```

等待看到：
```
VITE v5.x.x  ready in xxx ms

➜  Local:   http://localhost:5173/
```

---

### Step 3: 访问后台

1. 打开浏览器：http://localhost:5173/admin
2. 登录：
   - 用户名：`admin`
   - 密码：`Admin123456`
3. 点击左侧菜单：**3D大模型** → **专业版3D生成**

---

### Step 4: 测试生成流程

#### 4.1 上传图片
- 点击上传区域或拖拽图片
- 选择一张清晰的物品图片（建议：鞋子、杯子、玩具等）
- 或使用右侧"示例图片库"中的图片

#### 4.2 选择模型版本
- 选择：**hy-3d-3.0 标准版**
- 注意：极速版显示"即将开放"（已停用）

#### 4.3 开始生成
- 点击 **"🚀 开始生成"** 按钮
- 观察左侧进度条：
  ```
  [Mock] 正在生成3D模型...
  [Mock] 生成中... 10%
  [Mock] 生成中... 30%
  [Mock] 生成中... 50%
  [Mock] 生成中... 70%
  [Mock] 生成中... 90%
  [Mock] 生成完成！
  ```

#### 4.4 检查结果
- ✅ **成功标志**：
  - 进度条到达100%
  - 状态显示："生成完成！"
  - 右侧面板显示3D模型
  - 出现"⬇️ 下载GLB"按钮
  
- ❌ **失败标志**：
  - 进度条卡住不动
  - 状态显示错误信息
  - 右侧面板显示空白或错误提示

---

### Step 5: 验证3D模型

#### 5.1 基本交互
- **旋转**：鼠标左键拖动
- **缩放**：鼠标滚轮
- **平移**：鼠标右键拖动

#### 5.2 检查渲染
- 模型是否完整显示？
- 是否有光照效果？
- 是否可以流畅旋转？

#### 5.3 下载测试
- 点击 **"⬇️ 下载GLB"** 按钮
- 检查是否下载到本地
- 文件大小是否正常（应该 > 0 KB）

## 🔍 问题排查

### **问题1：右侧面板一直空白**

**检查步骤**：

1. **打开浏览器开发者工具**（F12）
2. **查看Console标签**：
   ```javascript
   // 应该看到类似日志
   [ProfessionalGeneration] Model generated successfully: http://localhost:8000/api/v1/experimental/download/hunyuan_cloud_xxx
   [ModelPreview] Model loaded successfully: http://localhost:8000/api/v1/experimental/download/hunyuan_cloud_xxx
   ```

3. **查看Network标签**：
   ```
   GET /api/v1/experimental/download/hunyuan_cloud_xxx
   Status: 200 OK
   Content-Type: model/gltf-binary
   Size: xxx KB
   ```

4. **如果Status不是200**：
   - 404：文件不存在
   - 500：服务器错误
   - 查看Response Body的错误信息

**解决方案**：
```bash
# 检查后端日志
tail -f backend/logs/app.log

# 手动查询任务状态
curl http://localhost:8000/api/v1/experimental/task/{task_id}

# 检查文件是否存在
ls -lh uploads/experimental/*.glb
```

---

### **问题2：进度条卡在某个百分比**

**可能原因**：
1. 后端任务异常
2. 网络连接中断
3. Mock模式未正确配置

**检查步骤**：
```bash
# 1. 查看后端日志
tail -f backend/logs/app.log | grep "hunyuan_cloud_"

# 2. 手动查询任务状态
curl http://localhost:8000/api/v1/experimental/task/{task_id}

# 响应示例
{
  "status": "completed",
  "progress": 100,
  "message": "[Mock] 生成完成！",
  "glb_path": "/path/to/model.glb"
}
```

**解决方案**：
- 如果状态是 `failed`：查看错误信息
- 如果状态是 `processing` 但长时间不变：重启后端
- 如果找不到任务：重新提交生成

---

### **问题3：显示"额度不足"**

**检查步骤**：
```bash
# 查询当前额度
curl -H "Authorization: Bearer YOUR_TOKEN" \
     http://localhost:8000/api/v1/quota/balance
```

**解决方案**：
1. 重置用户额度（需要数据库访问）
2. 或者修改代码临时增加额度

---

### **问题4：模型加载失败**

**错误提示**：
```
⚠️ 模型加载失败
请检查网络连接或稍后重试
```

**检查步骤**：
1. **Console标签**：
   ```javascript
   [ModelPreview] Failed to load model: Error: ...
   ```

2. **Network标签**：
   - 检查download请求的Status Code
   - 检查Response Headers的Content-Type

**解决方案**：
- 如果是404：文件不存在，检查后端
- 如果是CORS错误：检查后端CORS配置
- 如果是WebGL错误：检查浏览器支持
