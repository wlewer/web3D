## 📋 文档概述

本文档记录了首页（HomePage）使用的主要3D组件技术栈和实现方案，方便后续页面复用和组件调用。

## 🏠 首页架构概览

首页包含以下核心区域：

## 🎯 核心组件 1：UniversalGaussianCardV2

### 组件定位
**首页全屏3D展示的核心组件**，负责展示SPZ/Splat、GLB、PLY等多种格式的3D模型。

### 文件位置
```
src/web-frontend/src/components/3d/UniversalGaussianCardV2.tsx
src/web-frontend/src/components/3d/UniversalGaussianCardV2.css
```

### 技术栈

#### 1. 渲染引擎
- **Three.js** - 底层3D渲染库
  - `PerspectiveCamera` - 透视相机
  - `WebGLRenderer` - WebGL渲染器
  - `OrbitControls` - 轨道控制器（旋转、缩放）
  - `Box3` - 包围盒计算
  - `Vector3` - 三维向量运算

- **SparkRenderer** - 高斯泼溅专用渲染器
  - `SplatMesh` - Splat格式模型网格
  - 支持SPZ/Splat格式的GPU加速渲染

#### 2. 模型加载器
- **SplatMesh** - SPZ/Splat格式（高斯泼溅）
- **GLTFLoader** - GLB/GLTF格式（标准3D模型）
- **PLYLoader** - PLY格式（点云/网格）

#### 3. 动画系统
- **requestAnimationFrame** - 原生动画帧API
- **自定义缓动函数** - 二次缓入缓出（Quadratic In-Out）
- **lerpVectors** - 向量线性插值（相机平滑过渡）

#### 4. 状态管理
- **React Hooks** - `useState`, `useRef`, `useEffect`, `useCallback`, `useImperativeHandle`
- **状态机模式** - `UNINITIALIZED → INITIALIZING → READY → LOADING → LOADED → ERROR`

### 核心功能

#### 1. 智能居中算法
```typescript
smartFitCameraToObject(object, camera, canvas, config)
```
- **SplatMesh**: 动态计算真实包围盒 + margin系数
- **GLB模型**: 百分位裁剪空白区域 + 自动居中
- **PLY模型**: 区分点云/网格，自适应渲染

#### 2. 相机平滑过渡
```typescript
applyCameraConfig(config: CameraConfig)
```
- 使用 `requestAnimationFrame` + 自定义缓动函数
- `lerpVectors` 从起点到终点逐步插值
- 动画时长根据距离动态调整（800-1500ms）

#### 3. 场景装饰
- **粒子背景** - 3000个紫色渐变粒子
- **展示台** - 圆柱体平台 + 装饰环
- **产品标签** - 3D Sprite精灵（Canvas纹理）

#### 4. 鲁棒性机制
- **超时保护** - 25秒加载超时
- **心跳进度** - Vite dev server兼容
- **降级渲染** - Spark失败时回退到Three.js
- **帧同步锁** - 防止渲染冲突

## 🔧 技术实现细节

### 1. 相机平滑过渡实现

```typescript
// 核心算法：requestAnimationFrame + lerpVectors
const animate = (currentTime: number) => {
  const elapsed = currentTime - startTime;
  const progress = Math.min(elapsed / duration, 1);
  
  // 二次缓入缓出
  const easeProgress = progress < 0.5 
    ? 2 * progress * progress 
    : 1 - Math.pow(-2 * progress + 2, 2) / 2;
  
  // 线性插值
  camera.position.lerpVectors(startPosition, targetPosition, easeProgress);
  controls.target.lerpVectors(startTarget, targetTarget, easeProgress);
  
  if (progress < 1) {
    requestAnimationFrame(animate);
  }
};

requestAnimationFrame(animate);
```

## 📊 性能优化建议

### 1. 模型加载
- 使用SPZ格式（高斯泼溅）而非GLB
- 启用进度反馈和超时保护
- 避免同时加载多个大模型

### 2. 渲染优化
- 使用 `requestAnimationFrame` 而非 `setInterval`
- 限制FPS更新频率（1秒/次）
- 启用帧同步锁防止冲突

### 3. 内存管理
- 组件卸载时调用 `dispose()`
- 及时清理不再使用的模型
- 避免创建重复的场景对象

##  常见问题

### Q1: 模型加载后黑屏
**原因**: Canvas条件判断错误导致不显示  
**解决**: 简化为 `className={modelLoaded ? 'model-loaded' : ''}`

### Q2: 相机切换时跳跃
**原因**: 使用了Tween.js的update方法错误  
**解决**: 改用原生 `requestAnimationFrame` + `lerpVectors`

### Q3: 模型显示太小
**原因**: margin系数过大  
**解决**: SplatMesh使用 `margin * 0.7`，GLB使用 `margin * 0.8`

### Q4: 产品标签跳动
**原因**: 加载过程中标签可见  
**解决**: 初始隐藏标签，加载完成后显示

### Q5: GLB模型倒立
**原因**: 强制重置模型旋转信息（`object.rotation.set(0, 0, 0)`）  
**解决**: 移除强制重置代码，保留模型原始姿态（某些GLB文件自带旋转信息）  
**影响**: 仅影响GLB模型，SPZ/Splat模型不受影响
