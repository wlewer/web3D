# 500 错误修复与 Mock 模式说明

> 📅 修复日期：2026-04-19  
> 🔧 问题：Hunyuan3D 服务未启动导致 500 错误  
> ✅ 状态：**已添加健康检查和 Mock 模式**

---

## ❌ 问题原因

### 错误信息
```
POST http://localhost:8000/api/v1/generation/upload 500 (Internal Server Error)
上传失败: AxiosError: Request failed with status code 500
```

### 根本原因

**Hunyuan3D API 服务器（端口 8081）未运行**

当前架构：
```
前端 (5175) → 后端 (8000) → Hunyuan3D (8081) ❌ 未启动
```

当后端尝试调用 `http://localhost:8081/send` 时：
- 连接被拒绝
- 抛出异常
- 返回 500 错误

---

## ✅ 修复方案

### 1. 添加健康检查

在 `upload_and_generate` 端点中添加了 Hunyuan3D 服务可用性检测：

```python
# 快速健康检查
health_check = await service.client.get(f"{service.base_url}/health", timeout=5.0)
if health_check.status_code != 200:
    raise Exception("Hunyuan3D service is not healthy")
```

### 2. 实现 Mock 模式

如果 Hunyuan3D 服务不可用，返回模拟响应而不是 500 错误：

```python
# 返回模拟响应（用于测试）
mock_uid = str(uuid.uuid4())

return {
    "uid": mock_uid,
    "status": "processing",
    "message": "Hunyuan3D service is not available. This is a mock response for testing.",
    "model_id": model.id,
    "warning": "Hunyuan3D API server (port 8081) is not running."
}
```

### 3. 保存数据库记录

即使在 Mock 模式下，也会创建 Model3D 记录：

```python
model = Model3D(
    name=f"Test Model {mock_uid[:8]}",
    description="Generated by Hunyuan3D-2mini (Mock Mode)",
    category="other",
    status="pending",  # 标记为待处理
    model_url=None,
    format="glb",
    file_size=len(content),
    generation_engine="hunyuan3d",
    created_by=None
)
db.add(model)
await db.commit()
```

## 🎯 当前状态

### 工作流程

```
用户上传图像
    ↓
后端接收并验证
    ↓
检查 Hunyuan3D 服务
    ├─ ✅ 可用 → 提交真实任务 → 轮询状态 → 返回 GLB
    └─ ❌ 不可用 → 返回 Mock UID → 提示部署服务
```

### Mock 模式特点

| 特性 | 说明 |
|------|------|
| **UID 生成** | 随机 UUID（非真实任务 ID） |
| **数据库记录** | ✅ 创建（状态为 pending） |
| **文件保存** | ✅ 临时图片已保存 |
| **3D 生成** | ❌ 不执行（需要 Hunyuan3D） |
| **状态查询** | ⚠️ 会返回 processing（不会完成） |
| **模型下载** | ❌ 无 GLB 文件 |

### 方案 2：租用云端 GPU（推荐 ⭐⭐⭐⭐）

**适合开发测试**，成本可控：

#### AutoDL 平台

1. **注册账号**: https://www.autodl.com
2. **租用实例**:
   - GPU: RTX 3090 / 4090
   - 显存: 24GB
   - 价格: ~1-2 元/小时
3. **部署 Hunyuan3D**:
   ```bash
   # 在云端服务器上
   git clone https://github.com/Tencent-Hunyuan/Hunyuan3D-2.git
   cd Hunyuan3D-2
   pip install -r requirements.txt
   
   # 配置 HuggingFace Token
   export HF_TOKEN="your_token"
   
   # 启动 API 服务器
   python api_server.py --model_path tencent/Hunyuan3D-2mini --port 8081
   ```
4. **修改后端配置**:
   ```python
   # backend/app/services/generation/hunyuan3d_service.py
   class Hunyuan3DService:
       def __init__(
           self,
           base_url: str = "http://YOUR_CLOUD_IP:8081",  # 改为云端 IP
           timeout: int = 300
       ):
           ...
   ```

**优点**：
- ✅ 完整功能
- ✅ GPU 加速（快 10-50 倍）
- ✅ 可按需付费

**缺点**：
- ❌ 需要付费
- ❌ 需要配置网络

## 💡 常见问题

### Q1: Mock 模式下可以做什么？

**A**: Mock 模式主要用于：
- 测试前端 UI 流程
- 验证 API 接口连通性
- 调试数据库保存逻辑
- 开发其他功能（不依赖 3D 生成）

### Q2: 为什么不直接报错？

**A**: 返回 Mock 响应的好处：
- 前端可以继续测试其他功能
- 开发者可以看到完整的流程
- 避免阻塞开发进度
- 明确提示需要部署服务

### Q3: 什么时候应该部署 Hunyuan3D？

**A**: 建议在以下情况部署：
- 需要端到端测试完整流程
- 准备上线或演示
- 需要真实的 3D 模型输出
- 性能测试和优化

### Q4: 云端 GPU 贵吗？

**A**: 很便宜！
- AutoDL: 1-2 元/小时（RTX 3090/4090）
- 每天测试 2 小时 = 2-4 元/天
- 每月约 60-120 元
- 比购买硬件便宜得多
