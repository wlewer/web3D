# 3D环境贴图问题修复说明

## 问题描述

在访问后台管理系统时，出现以下错误：

```
Uncaught Error: Could not load studio_small_03_1k.hdr: Failed to fetch
```

这个错误来自 `@react-three/drei` 的 `Environment` 组件，该组件尝试从CDN加载HDR环境贴图文件。

## 根本原因

`Environment preset="studio"` 会尝试从以下URL加载HDR文件：
- `https://dl.polyhaven.org/file/ph-assets/HDRIs/hdr/1k/studio_small_03_1k.hdr`

在某些网络环境下（特别是国内网络），可能无法访问该CDN，导致加载失败。

## 已修复的文件

### 1. GenerationPage.tsx
**位置**: `src/web-frontend/src/admin/modules/experimental/pages/GenerationPage.tsx`

**修改内容**:
```tsx
// 修改前
<Environment preset="studio" />

// 修改后
{/* Environment组件可能导致HDR加载失败，暂时禁用 */}

<ambientLight intensity={0.6} />

<directionalLight position={[-10, -10, -5]} intensity={0.3} />
<pointLight position={[0, 5, 0]} intensity={0.5} />
```

## 如何重新启用Environment（可选）

如果您希望使用真实的环境贴图效果，有以下几种方案：

### 方案1：使用本地HDR文件（推荐）

1. 下载HDR文件到项目的 `public` 目录：
   ```bash
   # 从 Poly Haven 下载
   curl -o public/hdr/studio_small_03_1k.hdr \
     https://dl.polyhaven.org/file/ph-assets/HDRIs/hdr/1k/studio_small_03_1k.hdr
   ```

2. 修改代码使用本地文件：
   ```tsx
   import { Environment } from '@react-three/drei';
   
   // 使用本地文件
   <Environment files="/hdr/studio_small_03_1k.hdr" />
   ```

### 方案2：使用其他预设

某些预设可能使用不同的CDN或内置资源：
```tsx
<Environment preset="city" />  // 城市环境
<Environment preset="sunset" />  // 日落环境
<Environment preset="night" />  // 夜晚环境
```

### 方案3：添加错误边界

如果希望保留Environment但避免崩溃，可以添加错误边界：
```tsx
import { ErrorBoundary } from 'react-error-boundary';

<ErrorBoundary fallback={<FallbackLights />}>
  <Environment preset="studio" />
</ErrorBoundary>
```

其中 `FallbackLights` 是备用的光源配置。

这些光源配置可以在没有HDR环境贴图的情况下，仍然提供良好的3D视觉效果。

## 性能影响

禁用Environment组件对性能的影响：
- ✅ **正面影响**: 减少了网络请求和HDR文件加载时间
- ✅ **正面影响**: 降低了GPU渲染负担（不需要处理HDR环境映射）
- ⚠️ **轻微影响**: 缺少真实的环境反射效果（对于金属、玻璃等材质较明显）

对于大多数3D模型预览场景，当前的光源配置已经足够。
