## 五、创建文档流程

### 4.1 创建前检查

```markdown
□ 1. 确定文档属于哪个目录？
□ 2. 检查是否已有相同文档？
□ 3. 命名是否符合规范？
□ 4. 是否需要添加版本号？
□ 5. 根目录是否允许创建该文件？
```

### 4.2 正确位置

| 文件类型 | 正确位置 | 错误位置 |
|:---|:---|:---|
| 规范文档 | `docs/09-规范标准/` | 根目录 ❌ |
| 技术文档 | `docs/03-技术文档/` | 根目录 ❌ |
| 开发文档 | `docs/04-开发文档/` | 根目录 ❌ |
| 源代码 | `src/` | 根目录 ❌ |
| 配置文件 | `根目录` 或 `configs/` | 其他目录 ❌ |

## 六、常见问题

### 5.1 根目录可以创建什么？

```markdown
✅ 允许创建：
- README.md, CHANGELOG.md, LICENSE
- package.json, go.mod
- .gitignore, .env.example

❌ 禁止创建：
- 任何.md文档（除README系列外）
- 任何.html文件
- 任何临时文件
```

### 5.2 文档应该放在哪里？

docs/
