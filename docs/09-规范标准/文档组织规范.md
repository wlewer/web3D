## 二、文档命名规范

### 2.1 命名格式

```
{类型}-{标题}-v{主版本}.{次版本}.md

示例：
├── 需求-用户注册功能-v1.0.md
├── 设计-数据库架构-v1.2.md
├── 技术-3D渲染方案-v1.0.md
└── 规范-代码编写规范-v1.0.md
```

### 2.2 版本号规则

| 版本类型 | 规则 | 示例 |
|:---|:---|:---|
| 主版本 | 重大变更时更新 | v1.0 → v2.0 |
| 次版本 | 功能更新时更新 | v1.0 → v1.1 |
| 修订版 | Bug修复时更新 | v1.0 → v1.0.1 |

### 2.3 禁止的命名

```
❌ 错误示例：
├── 文档1.md
├── 新建文档.md
├── 技术文档最终版.md
├── 备份-需求-v1.0-2024.md
└── ~需求文档.docx
```

## 四、文档归档规则

### 4.1 归档条件

| 条件 | 说明 |
|:---|:---|
| 版本过期 | 新版本发布后，旧版本归档 |
| 内容重复 | 重复文档合并归档 |
| 项目结束 | 项目结束后的文档归档 |

### 4.2 归档流程

```
1. 识别需要归档的文档
2. 检查是否还有参考价值
3. 有价值 → 移动到 oldFile/
4. 无价值 → 删除
5. 更新相关引用
```

### 4.3 归档命名

```
oldFile/{原目录}/{类型}-{标题}-归档日期.md

示例：
oldFile/03-技术文档/技术方案-v1.0-归档-20260418.md
```

## 五、文档创建流程

### 5.1 创建前检查

```
□ 确定文档类型（对应目录）
□ 检查是否已有相同文档
□ 确定文档标题和版本号
□ 选择合适的模板
```

### 5.2 创建步骤

```
1. 选择正确的目录
2. 使用标准命名格式
3. 添加文档头部信息
4. 编写文档内容
5. 关联相关文档
```

### 5.3 创建清单

```markdown
## 文档创建清单

- [ ] 选择正确目录
- [ ] 使用标准命名
- [ ] 添加头部信息
- [ ] 编写目录结构
- [ ] 添加内容章节
- [ ] 添加版本历史
- [ ] 审核后发布
```

## 七、禁止事项

### 7.1 禁止在根目录创建

```
❌ 以下文件禁止在根目录创建：
├── 任何.md文档（除README系列外）
├── 任何.html文件
├── 任何.txt文件
├── 临时文件
└── 调试文件
```

### 7.2 禁止重复文档

```
❌ 同一内容不应出现多个文档：
├── 规范-代码规范-v1.0.md   ← 保留
└── 代码规范文档.md         ← 删除或归档
```

### 7.3 禁止随意命名

```
❌ 禁止以下命名：
├── 新建文档.md
├── 文档1.md
├── 备份.md
└── 临时.md
```

## 八、检查清单

### 8.1 文档创建检查

```markdown
创建新文档前：

□ 1. 确认文档属于哪个目录？
□ 2. 检查是否已有相同文档？
□ 3. 命名是否符合规范？
□ 4. 是否需要添加版本号？
□ 5. 文档头部信息是否完整？
□ 6. 内容结构是否清晰？
□ 7. 是否需要关联其他文档？
```

### 8.2 定期检查

```markdown
每月1日检查：

□ 文档总数是否合理（≤100）？
□ 是否有重复文档？
□ 是否有过期文档需要归档？
□ 目录结构是否清晰？
□ 命名是否规范？
```
