## 一、根目录允许文件

### 1.1 配置文件（必须）

| 文件名 | 类型 | 说明 |
|:---|:---|:---|
| `package.json` | JSON | 前端依赖配置 |
| `go.mod` | Go | Go模块配置 |
| `go.sum` | Text | Go依赖校验 |
| `.gitignore` | Text | Git忽略配置 |
| `.env.example` | Text | 环境变量模板 |

### 1.2 根目录文档（允许）

| 文件名 | 类型 | 说明 |
|:---|:---|:---|
| `README.md` | Markdown | 项目说明（必须） |
| `README.en.md` | Markdown | 英文说明 |
| `CHANGELOG.md` | Markdown | 变更日志 |
| `CONTRIBUTING.md` | Markdown | 贡献指南 |
| `FAQ.md` | Markdown | 常见问题 |
| `LICENSE` | Text | 开源协议 |
| `PROJECT_DELIVERY_CHECKLIST.md` | Markdown | 交付检查清单 |

### 1.3 根目录禁止文件

| 禁止类型 | 示例 | 说明 |
|:---|:---|:---|
| 临时文件 | `temp-*.txt`, `debug.md` | 必须删除 |
| 规范文档 | `*规范*.md`, `*总结*.md` | 必须归档到docs |
| 技术文档 | `*技术*.md`, `*方案*.md` | 必须归档到docs |
| HTML文件 | `*.html` | 必须归档到public |
| JS文件 | `*.js` | 必须归档到src |
| 日志文件 | `*.log` | 必须归档到logs |

## 四、文件名命名规范

### 4.1 根目录文件

```
✅ 正确示例：
├── README.md
├── CHANGELOG.md
├── package.json
└── .env.example

❌ 错误示例：
├── 项目说明.md
├── 技术文档.md
├── temp-readme.md
└── v1.0-README.md
```

### 4.2 命名规则

| 类型 | 规则 | 示例 |
|:---|:---|:---|
| 配置文件 | 英文小写 + 连字符 | `package.json`, `go.mod` |
| 文档文件 | 英文或中文 | `README.md`, `技术架构.md` |
| 禁止 | 中文+特殊字符 | `项目-说明-v1.0.md` ❌ |

---

## 五、检查机制

### 5.1 定期检查

```
每月1日自动检查根目录文件是否符合规范
```

### 5.2 检查清单

```bash
# 检查根目录文件数量
# 配置文件：≤10个
# 文档文件：≤10个
# 总文件数：≤25个
```

### 5.3 违规处理

| 违规类型 | 处理方式 |
|:---|:---|
| 临时文件 | 立即删除 |
| 规范文档 | 移动到 `docs/oldFile/` |
| HTML文件 | 移动到 `public/` |
| JS文件 | 移动到 `src/` |
