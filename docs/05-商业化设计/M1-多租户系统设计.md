### 1.2 请求流

```
Client → DNS (tenant.web3d.com) → Nginx (server_name ~^(?<tenant>.*)\.web3d\.com$)
     → 提取 tenant_code → 查询 Redis 缓存
     → 缓存未命中 → PostgreSQL tenants 表
     → 写入 request.state.tenant
     → 权限检查 (tenant.status == 'active')
     → 配额检查 (M4 Quota Gateway)
     → 业务处理 → 响应
```

### 2.1 核心表 Schema

```sql
-- tenants 表
CREATE TABLE tenants (
    id          SERIAL PRIMARY KEY,
    code        VARCHAR(64) NOT NULL UNIQUE,  -- 唯一标识，如 acme-corp
    name        VARCHAR(128) NOT NULL,
    status      VARCHAR(20) DEFAULT 'active', -- active, suspended, expired
    plan_id     INT REFERENCES plans(id),
    owner_id    INT REFERENCES users(id),
    db_schema   VARCHAR(64),  -- 物理隔离时使用
    settings    JSONB DEFAULT '{}',
    created_at  TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at  TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    deleted_at  TIMESTAMP NULL
);

-- tenant_configs 表
CREATE TABLE tenant_configs (
    id            SERIAL PRIMARY KEY,
    tenant_id     INT NOT NULL REFERENCES tenants(id) ON DELETE CASCADE,
    theme         JSONB DEFAULT '{"primaryColor":"#1677ff"}',
    features      JSONB DEFAULT '[]',  -- 启用功能白名单
    storage_limit BIGINT DEFAULT 1073741824,  -- 1GB
    api_rate_limit INT DEFAULT 1000,  -- 每小时
    custom_domain VARCHAR(255),
    created_at    TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at    TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

## 4. 中间件工作原理

### 4.1 三种租户解析方式

```python
# backend/app/core/tenant_middleware.py

class TenantMiddleware:
    def __init__(self, app):
        self.app = app

    async def __call__(self, scope, receive, send):
        request = Request(scope, receive)
        tenant = None

        # 方式①：子域名
        host = request.headers.get('host', '')
        subdomain = host.split('.')[0]
        if subdomain not in ['www', 'api', 'admin']:
            tenant = await self.get_tenant_by_code(subdomain)

        # 方式②：自定义 Header
        if not tenant:
            header_code = request.headers.get('x-tenant-code')
            if header_code:
                tenant = await self.get_tenant_by_code(header_code)

        # 方式③：Path 参数（API 场景）
        if not tenant:
            path_code = request.path_params.get('tenant_code')
            if path_code:
                tenant = await self.get_tenant_by_code(path_code)

        if not tenant:
            await send_error(send, 1002, "Tenant not found")
            return

        if tenant.status != 'active':
            await send_error(send, 2001, "Tenant is disabled")
            return

        request.state.tenant = tenant
        await self.app(scope, receive, send)
```

### 4.2 流程图

```
请求到达
    │
    ├── 解析 subdomain ──→ 命中？──→ 是 → 查询 Redis → 命中？──→ 是 → 注入 state
    │                       │否                 │否
    │                       │                   └── 查 PG → 缓存 Redis
    │                       │
    ├── 解析 Header x-tenant-code ──→ 命中？──→ 同上流程
    │                       │否
    │                       │
    └── 解析 Path param ──→ 命中？──→ 同上流程
                            │否
                            ▼
                        返回 401
```

## 6. 存储隔离方案（MinIO bucket 策略）

### 6.1 Bucket 命名规则

```
tenant-{tenant_code}-models      # 3D 模型文件
tenant-{tenant_code}-assets      # 图片、贴图
tenant-{tenant_code}-plugins     # 插件包
tenant-{tenant_code}-exports     # 导出文件
```

### 6.2 IAM Policy 模板

```json
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Action": ["s3:GetObject", "s3:PutObject", "s3:DeleteObject"],
      "Resource": "arn:aws:s3:::tenant-${tenant_code}-*/*",
      "Condition": {
        "StringEquals": {
          "s3:x-amz-meta-tenant-id": "${tenant_id}"
        }
      }
    }
  ]
}
```

### 6.3 隔离级别对比

| 级别 | 实现 | 适用场景 |
|------|------|----------|
| 共享 schema + 行级隔离 | `tenant_id` 字段过滤 | 标准 SaaS，成本最优 |
| 独立 schema | PostgreSQL schema 切换 | 企业级，合规要求 |
| 独立数据库 | 物理 DB 隔离 | 超大规模租户 |
| 独立 bucket | MinIO bucket 前缀隔离 | 存储层强制隔离 |

## 8. 部署配置

### 8.1 Nginx 域名路由

```nginx
# backend/nginx/nginx.conf

server {
    listen 80;
    server_name ~^(?<tenant>.+)\.web3d\.com$;

    location / {
        proxy_pass http://api_backend;
        proxy_set_header X-Tenant-Code $tenant;
        proxy_set_header Host $host;
    }
}

server {
    listen 443 ssl;
    server_name ~^(?<tenant>.+)\.web3d\.com$;

    ssl_certificate /etc/nginx/ssl/wildcard.web3d.com.crt;
    ssl_certificate_key /etc/nginx/ssl/wildcard.web3d.com.key;

    location / {
        proxy_pass http://api_backend;
        proxy_set_header X-Tenant-Code $tenant;
    }
}
```
