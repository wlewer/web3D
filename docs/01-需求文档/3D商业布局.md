一、官网设计思路 — 以 3DGS 为核心视觉语言
核心设计理念
市面上的官网都在卷动效、卷交互，但3DGS 直接展示三维模型这条赛道目前还是蓝海。设计上要突出"秒开即看"的体验：
plaintext
页面层级方案（从上到下）:
┌──────────────────────────────────┐
│  Hero 区：全屏 3DGS 模型环绕展示   │ ← 核心视觉冲击力
│  背景自动播放模型，支持鼠标拖拽     │
├──────────────────────────────────┤
│  产品/服务 区：网格模型卡片展示      │ ← 复用 V3 Grid layout

├──────────────────────────────────┤

设计要点
深色星空背景（已实现）— 凸显 3D 模型的发光质感，降低视觉干扰
轻交互 — 默认自动旋转 + 允许拖拽，让用户在沉浸状态下自然互动
极简 UI — 去掉一切冗余装饰，3D 模型本身就是最好的视觉元素
动态加载反馈 — 用渐入动画代替进度条（渐进式渲染天然优势）
二、这套技术适合的企业和商业场景
行业/场景	具体用途	V3 优势
电商 / 品牌官网（服饰/珠宝/数码/家居）	3D 商品展示，替代传统轮播图	用户可拖拽从各个角度查看商品，转化率可提升
数字展厅 / 艺术馆（博物馆、画廊）	文物/艺术品高精 3D 扫描展示	不需要用户下载 app，浏览器直接看
教育 / 科研（医学/生物/工程教学）	3D 教学模型交互展示	快速加载，学生端无需高端设备
建筑 / 室内设计（BIM、房产品牌）	设计方案 3D 展示，客户线上漫游	手机端即刻查看，无需安装软件
汽车 / 制造业（4S 店、配件展示）	车辆外观、配件 3D 预览	支持自定义颜色/配置实时切换
游戏 / 元宇宙（Web3 项目）	NFT 资产 3D 展示、虚拟物品预览	秒加载低门槛，提高用户留存
广告 / 营销（品牌活动页面）	3D 互动营销活动	区别于传统图文/视频，形成传播记忆点
最适合的企业画像： 拥有大量 3D 模型资产，需要在线展示但不希望用户安装任何软件、且对首屏体验要求高的品牌。
三、还需要做好哪些后台工作
当前后台已有：用户管理、模型管理、场景模板 CRUD、生成任务管理、配额管理。但距离一个完整的 CMS 系统还缺核心一环：
关键缺失：前台页面管理与布局引擎（最重要的）
目前 SceneTemplate 只有配置层面的管理，缺少页面布局的可视化管理。需要补充：
需求	说明	实现优先级
1. 页面管理	创建/编辑/发布多个独立页面（如首页/产品/关于）	P0
2. 区块管理（Section Builder）	每个页面可以增删拖拽多个 Section（Hero/网格/画廊/轮播/文字/CTA等），每个 Section 绑定一组模型参数	P0
3. 布局可视化配置	通过表单控制 V3 组件的所有 props（背景色、粒子、展示台、环绕模式、FOV等），实时预览	P0
4. 模型库管理	上传 SPZ/GLB 模型，自动缩略图，分类标签，拖拽排序	P1
5. 发布/回滚	页面版本控制，草稿→发布→回滚	P1
6. 数据埋点	每个 3D 模型的展示次数、交互时长、点击事件	P2
哪些工作可以复用现有代码
plaintext
现有代码 → 完全复用：
  - V3 组件体系（UniversalGaussianCardV3 / Base3DViewer）
  - 场景模板 API（templates.py CRUD）
  - 模型管理 API（models.py）
  - 权限系统（auth.py + dependencies.py）

需要新增（后端模型 + API）：
  - Page（页面表）：标题、路径、SEO信息、发布的区块配置
  - PageSection（区块表）：关联 Page，包含 SectionType + 配置 JSON
  - SectionType 枚举：hero / grid / gallery / carousel / text / divider...

需要新增（前端管理界面 + 预览）：
  - Admin H5 页面编辑器（拖拽区块 + 配置表单 + 实时预览）
  - 页面发布管理列表
  - 模型上传/管理界面
