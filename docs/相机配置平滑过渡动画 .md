# 相机配置平滑过渡动画 - 技术实现文档

## 🎯 问题解决

### 原问题
当加载已保存的相机配置时，相机会**瞬间跳转**到目标位置，产生明显的跳跃动作，用户体验不佳。

### 解决方案
使用 **Tween.js** 动画库实现相机位置、观察点和缩放的**平滑过渡动画**，消除跳跃感。

---

## ️ 技术实现

### 1. 安装依赖

```bash
npm install @tweenjs/tween.js
```

### 2. 导入Tween.js

```typescript
import { Tween, Easing } from '@tweenjs/tween.js';
```

### 3. 核心实现原理

#### **3.1 动画流程**

```
起始位置（智能居中算法计算）
    ↓
计算距离 → 动态确定动画时长（600ms-1200ms）
    ↓
Tween动画（缓入缓出）
    ↓
    每一帧：lerpVectors线性插值
    ↓
目标位置（保存的相机配置）
```

#### **3.2 代码实现**

**位置A：自动应用自定义配置（useEffect）**

```typescript
// 保存起始位置
const startPosition = camera.position.clone();
const startTarget = controls.target.clone();
const startZoom = camera.zoom;

// 目标位置
const targetPosition = new THREE.Vector3(
  config.position[0],
  config.position[1],
  config.position[2]
);
const targetTarget = new THREE.Vector3(
  config.target[0],
  config.target[1],
  config.target[2]
);
const targetZoom = config.zoom;

// 计算相机移动距离，根据距离动态调整动画时长
const distance = startPosition.distanceTo(targetPosition);
const duration = Math.min(Math.max(distance * 500, 600), 1200); // 600ms-1200ms

// 使用Tween.js实现平滑过渡
const tweenData = { t: 0 };

new Tween(tweenData)
  .to({ t: 1 }, duration)
  .easing(Easing.Quadratic.InOut) // 缓入缓出
  .onUpdate(() => {
    // 插值计算相机位置
    camera.position.lerpVectors(startPosition, targetPosition, tweenData.t);
    
    // 插值计算控制器target
    controls.target.lerpVectors(startTarget, targetTarget, tweenData.t);
    controls.update();
    
    // 插值计算缩放
    camera.zoom = startZoom + (targetZoom - startZoom) * tweenData.t;
    camera.updateProjectionMatrix();
  })
  .onComplete(() => {
    console.log('✅ 自定义相机配置平滑过渡完成:', config);
  })
  .start();

// 在动画循环中更新Tween
const animate = () => {
  if (tweenData.t < 1) {
    requestAnimationFrame(animate);
  }
};
animate();
```

**位置B：loadCameraConfig方法（手动加载）**

同样的实现逻辑，确保通过ref调用的loadCameraConfig方法也有平滑过渡。

## 📊 动画参数详解

### 1. 动态时长计算

```typescript
const distance = startPosition.distanceTo(targetPosition);
const duration = Math.min(Math.max(distance * 500, 600), 1200);
```

**逻辑说明**：
- 计算起始位置和目标位置的欧几里得距离
- 基础速率：每单位距离500ms
- 最小值：600ms（即使距离很近也要有可见的过渡）
- 最大值：1200ms（距离再远也不会超过1.2秒）

**示例**：
- 距离 = 0.5 → 时长 = 600ms（最小值）
- 距离 = 1.5 → 时长 = 750ms
- 距离 = 3.0 → 时长 = 1200ms（最大值）

### 2. 缓动函数

```typescript
.easing(Easing.Quadratic.InOut)
```

**Quadratic.InOut**：二次方缓入缓出
- **缓入（0-50%）**：速度从慢到快，避免突然启动
- **缓出（50%-100%）**：速度从快到慢，自然停止

**效果**：符合物理运动规律，用户体验自然流畅

### 3. 线性插值（Lerp）

```typescript
camera.position.lerpVectors(startPosition, targetPosition, tweenData.t);
```

**原理**：
```
当前值 = 起始值 + (目标值 - 起始值) × t
```

其中 `t` 从 0 平滑过渡到 1

**三个插值目标**：
1. **相机位置**：`camera.position` - 三维向量
2. **观察点**：`controls.target` - 三维向量
3. **缩放级别**：`camera.zoom` - 标量值

## 🔧 性能优化

### 1. 动画循环优化

```typescript
const animate = () => {
  if (tweenData.t < 1) {
    requestAnimationFrame(animate);
  }
};
```

**优化点**：
- 使用 `requestAnimationFrame` 而非 `setInterval`
- 自动跟随屏幕刷新率（通常60fps）
- 动画完成后立即停止循环，不浪费性能

### 2. 避免不必要的更新

```typescript
if (tweenData.t < 1) {
  requestAnimationFrame(animate);
}
```

**逻辑**：
- 只在动画进行中调用 `requestAnimationFrame`
- 动画完成（t=1）后自动停止
- 不产生额外的渲染开销

### 3. 单次动画

- 每次配置变化只触发一次动画
- 不会累积多个动画队列
- 避免内存泄漏
