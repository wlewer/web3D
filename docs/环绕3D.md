# 3D模型环绕播放方案 - 设计文档

##  需求概述

为3D模型展示大屏看板设计自动环绕播放功能，支持多种视角切换和动态效果，提升展示震撼度。

## 🎨 环绕播放效果方案

### 方案1：基础环绕模式 (推荐)

**效果**：相机围绕模型中心点做水平360°环绕运动

**参数**：
- 环绕半径：固定或可调
- 高度：固定或缓慢上下浮动
- 速度：可配置（慢/中/快）
- 方向：顺时针/逆时针

**适用场景**：
- 产品展示
- 艺术品展示
- 基础轮播展示

**实现方式**：
```typescript
interface OrbitConfig {
  radius: number;           // 环绕半径
  height: number;           // 相机高度
  speed: number;            // 旋转速度（度/秒）
  direction: 'clockwise' | 'counter-clockwise';
  autoTarget: boolean;      // 自动对准模型中心
}
```

---

### 方案2：螺旋环绕模式

**效果**：相机在环绕的同时上下移动，形成螺旋轨迹

**参数**：
- 螺旋圈数：1-3圈
- 高度范围：最低点到最高点
- 速度：可配置

**适用场景**：
- 建筑模型展示
- 复杂结构展示
- 震撼效果展示

**实现方式**：
```typescript
interface SpiralConfig {
  radius: number;
  minHeight: number;        // 最低高度
  maxHeight: number;        // 最高高度
  loops: number;            // 螺旋圈数
  speed: number;
  direction: 'up' | 'down'; // 螺旋方向
}
```

---

### 方案3：多视角切换模式 (大屏推荐)

**效果**：在多个预设视角之间平滑切换，每个视角停留一定时间

**预设视角**：
1. **正面特写** - 展示主体
2. **45度角** - 展示立体感
3. **俯视全景** - 展示整体布局
4. **侧面细节** - 展示侧面特征
5. **低角度仰视** - 震撼视角

**参数**：
- 视角列表：可配置多个视角
- 停留时间：每个视角显示时长
- 过渡时间：视角切换的动画时长

**适用场景**：
- 大屏看板展示
- 展厅演示
- 自动轮播

**实现方式**：
```typescript
interface MultiViewConfig {
  views: CameraConfig[];    // 预设视角列表
  duration: number;         // 每个视角停留时间（秒）
  transitionDuration: number; // 过渡动画时长（毫秒）
  loop: boolean;            // 是否循环播放
  pauseOnInteraction: boolean; // 用户交互时暂停
}
```

---

### 方案4：动态聚焦模式

**效果**：相机在环绕的同时，动态改变观察焦点（从整体到局部细节）

**参数**：
- 焦点切换序列：整体 → 局部1 → 局部2 → 整体
- 每个焦点的停留时间
- 过渡动画

**适用场景**：
- 复杂产品细节展示
- 教育演示
- 交互式导览

**实现方式**：
```typescript
interface FocusConfig {
  sequence: Array<{
    position: [number, number, number];
    target: [number, number, number];
    zoom: number;
    duration: number;       // 停留时间
  }>;
  transitionDuration: number;
  loop: boolean;
}
```

---

### 方案5：混合模式 (终极方案)

**效果**：结合以上多种模式，按时间段自动切换不同效果

**时间段划分**：
- 0-10秒：水平环绕（基础展示）
- 10-20秒：螺旋上升（整体展示）
- 20-35秒：多视角切换（细节展示）
- 35-45秒：动态聚焦（重点展示）
- 45-60秒：回到初始位置，循环

**适用场景**：
- 高端展厅
- 重要客户演示
- 展会展示

**实现方式**：
```typescript
interface HybridConfig {
  modes: Array<{
    type: 'orbit' | 'spiral' | 'multiview' | 'focus';
    config: OrbitConfig | SpiralConfig | MultiViewConfig | FocusConfig;
    duration: number;       // 该模式持续时间
  }>;
  loop: boolean;
}
```

## 💻 技术实现方案

### 1. 核心架构

```
CameraAnimationController
├── AnimationEngine (动画引擎)
│   ├── Tween.js (平滑过渡)
│   ├── Timeline (时间轴管理)
│   └── KeyframeSystem (关键帧系统)
├── ModeManager (模式管理器)
│   ├── OrbitMode (环绕模式)
│   ├── SpiralMode (螺旋模式)
│   ├── MultiViewMode (多视角模式)
│   └── FocusMode (聚焦模式)
├── ConfigManager (配置管理器)
│   ├── PresetConfigs (预设配置)
│   ├── UserConfigs (用户自定义)
│   └── Storage (localStorage/后端API)
└── UIController (UI控制器)
    ├── ModeSelector (模式选择器)
    ├── SpeedControl (速度控制)
    ── TimelineControl (时间轴控制)
```

### 3. 动画引擎实现

```typescript
class CameraAnimationEngine {
  private camera: THREE.PerspectiveCamera;
  private controls: OrbitControls;
  private tweens: Map<string, Tween>;
  private timeline: Timeline;
  
  // 平滑过渡到目标视角
  transitionTo(target: CameraConfig, duration: number): Promise<void> {
    return new Promise((resolve) => {
      const tweenData = { t: 0 };
      
      new Tween(tweenData)
        .to({ t: 1 }, duration)
        .easing(Easing.Quadratic.InOut)
        .onUpdate(() => {
          this.camera.position.lerpVectors(
            this.camera.position,
            new THREE.Vector3(...target.position),
            tweenData.t
          );
          this.controls.target.lerpVectors(
            this.controls.target,
            new THREE.Vector3(...target.target),
            tweenData.t
          );
          this.controls.update();
        })
        .onComplete(resolve)
        .start();
    });
  }
  
  // 环绕动画
  orbitAnimation(config: OrbitConfig): void {
    // 使用三角函数计算相机位置
    // x = radius * cos(angle)
    // z = radius * sin(angle)
    // y = height
  }
  
  // 螺旋动画

    // y = minHeight + (maxHeight - minHeight) * (angle / (2π * loops))

}
```

## 🎯 推荐方案

### 对于大屏看板展示

**推荐：方案3（多视角切换）+ 方案5（混合模式）的组合**

**理由**：
1. **视觉冲击力强** - 不同视角切换比单一环绕更有层次感
2. **可控性强** - 可以精确控制每个视角的展示重点
3. **易于配置** - 用户可以自定义视角序列
4. **兼容性好** - 可以与其他模式组合使用

**配置示例**：
```typescript
const dashboardConfig: HybridConfig = {
```

## 🔧 技术要点

### 1. 性能优化

- 使用`requestAnimationFrame`确保流畅动画
- 限制最大帧率到60fps
- 用户交互时暂停动画，减少不必要的计算
- 使用`THREE.Vector3.lerpVectors`进行插值，避免创建新对象

### 2. 用户体验

- 提供平滑的过渡动画（缓入缓出）
- 用户交互时自动暂停，2秒后恢复
- 提供暂停/播放按钮
- 显示当前播放进度

### 3. 兼容性

- 支持所有现有模型格式（SPZ、GLB）
- 兼容现有的相机配置保存功能
- 不破坏现有功能，作为可选增强

### 4. 可扩展性

- 配置系统支持JSON导入/导出
- 可以添加新的动画模式
- UI组件可以独立使用
